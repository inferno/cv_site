# CV - Константин Савельев

Резюме с премиальным дизайном на HTML/CSS.

## Запуск с Docker

### Быстрый старт

**Важно:** Для production (SSL) убедитесь, что:
1. DNS для домена `ksavelyev.ru` настроен и указывает на ваш сервер
2. Порты 80 и 443 открыты

Просто запустите:

```bash
docker-compose up -d --build
```

При первом запуске контейнер **автоматически**:
- Создаст необходимые директории
- Загрузит рекомендуемые параметры TLS
- Создаст временный самоподписанный сертификат
- Запросит настоящий SSL сертификат от Let's Encrypt для `ksavelyev.ru`
- Настроит автообновление сертификатов

**Домен:** ksavelyev.ru, www.ksavelyev.ru
**Email:** i@ksavelyev.ru

Сертификаты автоматически обновляются каждые 12 часов.
Nginx автоматически перезагружается каждые 6 часов для применения обновлений.

### Остановка

```bash
docker-compose down
```

### Перезапуск после изменений

```bash
docker-compose restart
```

## Troubleshooting

### Ошибка получения сертификата

Если контейнер не может получить сертификат:

1. Убедитесь, что DNS настроен правильно:
```bash
nslookup ksavelyev.ru
```

2. Проверьте доступность портов:
```bash
sudo netstat -tulpn | grep :80
sudo netstat -tulpn | grep :443
```

3. Проверьте логи контейнера:
```bash
docker-compose logs nginx
```

**Примечание:** Если сертификат не удалось получить, контейнер продолжит работу с самоподписанным сертификатом (90 дней).

### Перевыпуск сертификата

Если нужно перевыпустить сертификат:

1. Остановите и удалите контейнеры:
```bash
docker-compose down -v
```

2. Запустите заново:
```bash
docker-compose up -d --build
```

Контейнер автоматически получит новый сертификат.
